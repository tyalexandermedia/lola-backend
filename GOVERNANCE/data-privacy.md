# LOLA OS Data Privacy And Client Separation

## Principle

Client data must remain separated by slug, tenant, account, or source system. Reusable intelligence should contain conclusions and evidence references, not raw private data.

## Do Not Store

- API keys
- Admin keys
- Bearer tokens
- Passwords
- Raw call recordings
- Raw PII exports
- Private customer lists

## Safe To Store

- Public URLs
- Non-secret client configuration
- Service lists
- Service areas
- SEO target queries
- AI visibility prompts
- Proof status
- Integration status

## Promotion Rule

Before client-specific evidence becomes blueprint-level intelligence, remove private data and clearly mark the claim status.
