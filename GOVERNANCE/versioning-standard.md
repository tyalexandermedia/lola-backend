# LOLA OS Versioning Standard

LOLA OS versions reusable systems, not every note.

## Versioned Objects

- Patterns
- Blueprints
- Products
- Prompts
- Agents
- Templates
- Playbooks
- Automations

## Version Format

Use semantic versions:

- `1.0.0`: first usable version.
- `1.1.0`: additive evidence, services, workflow, or client validation.
- `2.0.0`: materially stronger or incompatible model.

## Example

- `Pressure Washing Blueprint v1.0.0`: Sandbar only.
- `Pressure Washing Blueprint v1.1.0`: Sandbar plus Tampa Bay Power Clean.
- `Pressure Washing Blueprint v2.0.0`: proven across five or more businesses or markets.

## Rule

If a change affects client generation, tracking expectations, proof requirements, or repeatable deployment behavior, version it.
