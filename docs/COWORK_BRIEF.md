## Tech stack
- **Frontend:** Vite + React SPA (client-rendered) in `lola-backend/frontend/`, deployed on
  **Vercel**. Build = `tsc && vite build && node scripts/prerender.mjs` (prerender writes
  static HTML for marketing routes so AI crawlers see real content; `/lp/*` excluded).
- **Backend:** FastAPI (`main.py`) on **Railway** (`lola-backend-production.up.railway.app`),
  SQLite via aiosqlite. Frontend proxies `/lead-gen`, `/t/*` etc. to it via `vercel.json`.
- **Repos:** `tyalexandermedia/lola-backend` (app + FastAPI + frontend) and
  `tyalexandermedia/lola-seo` (separate static marketing site).
- **Payments:** Stripe Payment Links + webhook. **Email:** Resend. **SMS/voice:** Twilio.
  **AI:** Anthropic API. **Analytics:** PostHog.

## Ways of working
- **Branch:** `claude/lola-two-tier-pricing-8x0sbz`. After a PR merges, RESET the branch onto
  the new main (`git fetch origin main && git checkout -B <branch> origin/main`) so new work
  is fresh, not stacked on merged history.
- **Flow:** build → verify (`npm run build`, `python3 -m py_compile`, unit tests) → commit →
  push → open PR → merge → Vercel/Railway auto-deploy.
- **DEPLOY GAP (important):** nothing shows on the live site until it's **merged to `main`** —
  a pushed branch alone does nothing in production. If Ty "doesn't see it," it's almost always
  an unmerged branch.
- Commit trailer: `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`. Set
  `git config user.email noreply@anthropic.com` so commits aren't flagged unverified.
- Cadence: Ty often says "complete next best ROI" — pick the highest-value item, ship it end
  to end (code → verify → PR → merge), then report + suggest the next.
